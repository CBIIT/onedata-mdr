caDSR II Release 1.64.2.0 includes the following improvements::

    New Features

    • Added APIs to return Context Names and Workflow Status List of Values.
    • Added Change Management email notifications for versioned content.
    • Enhanced Code Map Model Mapping View to make all nodes visible to non-logged-in customers.

    Bug Fixes

    • Resolved REST API issue where Data Element API incorrectly returned DECs.
    • Fixed Bulk Load issues related to DEC updates and form imports.
    • Corrected multiple Model Mapping and Model Mapping Import bugs, including:
      • Incorrect characteristic rule handling.
      • Missing semantic equivalency.
    • Fixed download issues with CDE JSON format and Model Mapping Rules.

    Improvements

    • Enhanced CDE Match usability by allowing users to select permissible values to include/exclude when matching.
    • Strengthened validation in Code Map Services for mapping consistency.
    • Improved UI and functionality for Model Mapping, including:
      • Pseudocode generation.
      • Transformation rule UI.
      • Value mapping imports.
    • Refined External Terminology Crosswalk with updated labels and column headings.
    • Expanded search and browsing capabilities, including UI changes and export enhancements.

    This update focuses on improving model mapping rule validation, enhancing user experience, fixing critical bugs, and strengthening system compliance and interoperability.

    As always, this release also has a number of UI improvements and bug fixes. For details, refer to the caDSR II Release Notes.

    caDSR II provides access to NCI's metadata standards for: Common Data Elements (CDEs), Case Report Forms (CRFs), and Data Models. These metadata resources describe a common way of structuring and understanding data.

    The "Links to Favorites" section Data Elements - View/Browse provides access to NCI's CDEs and Forms provides access to our CRFs. It also includes links for browsing or searching specific collections of content using various "Child Filters".

    After finding content by searching, you have multiple export options:

    • Export the displayed content using Delivery Options.
    • Add the displayed content to the User Cart, use Download Collection to build a collection (a custom set of content), and request a download of that collection in various formats:
      • Legacy caDSR Excel and XML formats
      • Medidata Rave ALS
      • REDCap Data Dictionaries
